# ai-forum

一个参考百度贴吧（Tieba）核心体验的社区论坛方案。项目目标是在保留“吧文化”和社区自治氛围的前提下，引入**实用且安全的 AI 辅助能力**，提升内容发现、互动效率与治理质量。

> 当前 README 聚焦**产品方案与页面设计**（What & Why），不涉及具体技术选型或实现细节（How）。
> 项目整体以 AI 为核心驱动力，覆盖从方案设计到代码实现的全流程。

---

## 1. 项目定位（Product Positioning）

### 1.1 核心愿景
- 以「吧（Bar）」作为兴趣社区的基本单元，形成清晰的圈层结构。
- 提供低门槛、快节奏的发帖与回复体验，强化“讨论现场感”。
- 用 AI 做“辅助决策”而非“替代社区”，尊重用户表达与版主管理。

### 1.2 目标用户
- 有明确兴趣圈层需求的普通用户（浏览、发帖、互动）。
- 吧务与版主团队（维护秩序、引导内容、组织活动）。
- 平台管理者（跨吧治理、风控、生态健康运营）。

---

## 2. v1 产品目标 / Goals

1. **Tieba-style structure**  
   以吧为主入口，每个吧拥有独立主页、内容流、规则与管理角色。

2. **Fast community interaction**  
   低摩擦发布与回复：快速发帖、楼层回复、@提及、轻量互动。

3. **AI-assisted practical features**  
   仅上线可解释、可控的 AI 场景：  
   - 重复帖检测与相关帖推荐  
   - 基础内容审核辅助（垃圾/攻击/违规提示）  
   - 长帖/长串讨论搜索与摘要

4. **Scalable product foundation**  
   从产品信息架构上预留角色权限、治理流程和可观测运营指标的扩展空间。

---

## 3. MVP 范围（v1）/ 第一版范围

### 3.1 Core Community / 核心社区能力

#### A. Bars（吧）
- 创建吧（名称、简介、头像、吧规）
- 编辑吧资料（权限控制：吧主/版主）
- 吧分类与标签（用于发现页聚合）
- 吧关注/加入（加入后可在“我的吧”查看）
- 吧务角色（基础模型）：吧主 / 版主 / 普通成员

#### B. Posts（帖子）
- 发帖、编辑、删除（按权限约束）
- 帖子列表：按“最新 / 热门”切换
- 帖子详情：标题、正文、发布时间、作者信息、互动区
- 基础互动：点赞、收藏、分享（轻量社交）

#### C. Replies（回复）
- 楼层式回复（主回复按时间或热度展示）
- 支持 reply-to（对某楼层进行二级回复）
- 可选引用回复（在楼层中引用前文）
- @用户提醒（基础通知能力）

#### D. Accounts / 身份系统
- 注册 / 登录
- 基础个人主页（昵称、头像、签名、发帖与回复记录）
- 基础角色与权限：用户 / 版主 / 管理员

## 4. 信息架构与页面设计（参考百度贴吧）

> v1 页面重点强调“贴吧式高频浏览 + 快速回帖”体验。

### 4.1 全局导航结构
- 顶部导航：首页、发现、我的吧、消息、个人中心
- 全局搜索：支持吧名、帖子关键词、用户昵称
- 快速发布入口：始终可见（移动端悬浮按钮 / PC 顶部按钮）

### 4.2 关键页面清单

1. **首页（推荐流）**
   - 展示关注吧的最新内容 + 平台推荐内容
   - 提供“最新 / 热门 / 关注吧”切换
   - 可直接进入帖子详情或吧主页

2. **发现页（吧广场）**
   - 按分类、标签、热度发现吧
   - 支持新吧冷启动曝光（新建吧推荐区）
   - 展示吧基础信息：头像、简介、成员数、今日发帖数

3. **吧主页（Bar Home）**
   - 吧头区：吧名、简介、吧规摘要、加入按钮
   - 吧内 feed：最新/热门帖子列表
   - 侧边/抽屉信息：吧务成员、公告、精选帖入口
   - 吧内发帖入口固定可达

4. **发帖页（Create Post）**
   - 标题 + 正文输入
   - 发布前提示：相关帖推荐、潜在重复提醒
   - 吧规风险提示（如敏感词、灌水提醒）但不强制替用户改写

5. **帖子详情页（Thread Detail）**
   - 主楼 + 楼层回复区（贴吧风格核心）
   - 楼层支持回复、引用、@、点赞
   - 长串讨论提供“本帖搜索”和“AI 摘要”入口

6. **消息页（Notifications）**
   - @我、回复我、系统通知、吧务通知分组
   - 重点提醒可一键回到对应楼层

7. **个人中心**
   - 我的发帖 / 回复 / 收藏
   - 我关注的吧
   - 基础资料维护

### 4.3 交互原则
- **低门槛**：优先保证“看帖快、回帖快、切换吧快”。
- **强上下文**：回复时明确显示“回复对象/楼层引用”。
- **轻打断**：AI 提示以“建议卡片”呈现，默认不阻断发布流程（除明确违规）。

## 5. AI 辅助能力设计（v1）

### 5.1 重复检测与相关帖推荐
- 场景：用户发帖前 / 发帖中
- 目标：减少重复提问，提高内容沉淀效率
- 呈现方式：推荐“可能相关帖子”列表，用户可选择继续发布或跳转阅读

### 5.2 基础审核辅助
- 场景：发帖、回帖提交阶段
- 目标：降低垃圾信息、攻击性内容与明显违规内容
- 呈现方式：风险级别提示 + 对应原因标签（可解释）
- 原则：AI 负责提示与建议，最终处置由规则和人工治理体系兜底

### 5.3 长线程搜索与摘要
- 场景：超长帖子/高楼层讨论
- 目标：让后来者快速理解上下文并定位关键信息
- 呈现方式：  
  - “本帖内搜索”快速定位楼层  
  - “讨论摘要”提炼核心结论、争议点与常见解答

## 6. 角色与治理模型（产品视角）

### 6.1 角色分层
- **普通用户**：发帖、回帖、关注吧、举报内容
- **版主/吧务**：吧内内容管理、公告维护、规则执行
- **管理员**：跨吧治理、违规申诉处理、平台级策略维护

### 6.2 基础治理流程（v1）
- 用户举报 → 吧务初审 → 平台复核（必要时）
- AI 风险提示进入待处理队列，辅助人工优先级排序
- 对被处置内容提供基础原因说明，降低治理黑箱感

---

## 7. v1 边界说明（明确不做）

- 不做复杂社交关系链（如完整关注图谱推荐系统）
- 不做高度自动化的“AI 代发/代聊”
- 不做重度创作工具（复杂排版、富媒体工作流等）
- 不做跨平台超级运营后台（先保证核心社区闭环）

---

## 8. 成功标准（验收导向）

### 8.1 社区活跃
- 新吧创建量、吧内发帖率、回复率、次日回访率

### 8.2 内容质量
- 重复帖占比下降、有效回复占比提升、违规内容处理时效

### 8.3 使用体验
- 发帖成功率、回复成功率、长帖阅读完成率
- AI 建议采纳率（相关帖点击、摘要使用、风险提示修正率）

---

## 9. 一句话总结

v1 以“贴吧式社区结构 + 高频互动体验”为核心，AI 仅作为可解释、可控的辅助层，帮助社区更高效地发现内容、维护秩序并沉淀讨论价值。
